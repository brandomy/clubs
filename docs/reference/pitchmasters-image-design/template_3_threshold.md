# Pitchmasters Visual Template 3: THRESHOLD (Filtered Signal Crossing Boundary)

You are generating a Pitchmasters visual under the Pitchmasters Visual Charter.
All charter rules are mandatory.

## CANONICAL METAPHOR
Threshold — Filtered Signal Crossing a Boundary

## PURPOSE
Represent discrete transformation from ambiguity to clarity.

## SUBJECT
A single signal form interacting with a clearly defined boundary.
Only one signal state may pass the boundary.

## COMPOSITION
- One boundary element
- One signal form
- No more than 3 total elements
- Strong directional logic without arrows

## FORM LANGUAGE
- Abstract signal representation
- Clean separation of states
- No gradients on objects

## COLOR
- Background color consistent across image
- Pre-threshold and post-threshold contrast
- One accent maximum

## PROHIBITED
- Light beams, glow effects
- Screens or filters resembling UI
- Optical metaphors (lenses, prisms)

## GOAL CHECK
The transition must appear inevitable, not dramatic.
